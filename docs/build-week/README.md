# Build Week Foundation

**Status:** Active — Documentation Only

## Required first document

Every future Build Week session must begin with the single authoritative operational handoff:

- [Build Week Operational Context](./CONTEXT.md)

Read `CONTEXT.md` completely before opening other Build Week files. It identifies the current sprint, latest completed work, locked references, evidence inventory, priorities, blockers, next authorized action, founder intent, and exact startup prompt. This README indexes the detailed working set; it does not replace the operational handoff.

## Purpose

This directory preserves working material for SCZN3's Build Week submission. It captures discoveries, narrative structure, evidence needs, and possible presentation language before they are polished into submission copy.

Build Week is an unfinished checkpoint. This workspace must not imply that SCZN3, its collaboration method, or its documentation is complete.

## Working rule

> Preserve ideas before polishing.

The material here is not marketing approval, architecture, production documentation, or final manuscript prose. Category language, slogans, product descriptions, quotations, and causal claims remain provisional until their evidence and publication status are reviewed.

## Evidence boundaries

- Founder recollection remains identified as founder recollection until corroborated.
- Exact quotations retain their wording and provenance.
- Architecture documents establish technical claims; this folder does not.
- Production claims require production evidence.
- Product-category language remains **Needs Validation** until reviewed.
- Unpublished phrases remain **Working Language**.
- Conversation excerpts are primary evidence only after their wording, speaker, source location, and publication approval are verified.
- A working conversation summary must never be presented as a verbatim transcript.

## Maintenance

1. Capture new ideas in the parking lot or narrative notes.
2. Link the strongest available discovery, episode, milestone, ADR, or production evidence.
3. Move material into the outline only when its role in the submission is understood.
4. Do not convert working language into an approved claim without founder review.
5. Preserve earlier meaning when revising; do not erase the path by which the idea developed.
6. Weave approved conversation evidence through the narrative; do not bury the invention process in an appendix.
